    Internet ✶⋆.˚

    What it the internet?

    They are submarine cables connected to serves located in data centres.

    The internet is a global network of interconnected computers, which enables the exchange of data and information between devices anywhere in the world, through standdardised communication protocols, mainly TCP/IP.

    Origin of the Internet

    The internet was created from ARPANET, developed in the United States in the late 1960s. Its original motivation was military: to ensure continuity of communication and acess to strategic information even in the event of attacks, such as a nuclear strike. Today, its use is vastly different, aimed at civilian and commercial purposes.

    The internet as we know it today only emerged in the 1980s-1990s.

    ➜ ARPANET (Advanced Research Projects Agency Network): was the first packet-switching computer network in history and is considered the origin of the internet.

    Most Common Terms

    ➜ Datacentre: is a physical facility designed to house critical computing systems, such as:

    • Servers (high-perfomance computers);
    • Storage devices (HDs, SSDs);
    • Network equipment (routers, switches, firewalls);
    • Fontes de energia redundante e sistemas de resfriamento;
    • Main function: To store, process, and distribute data;
    • A datacentre is the physical heart of the internet. It's where the computers that store websites, apps, databases,email, videos, and al digital information circulatingglobally are located;
    • Examples of companies operating datacentres: Google, Amazon (AWS), Microsoft (Azure), Meta, IBM, Oracle.

    ➜ IP (Internet Protocol): is the protocol that enables locating, identifying and sending between devices on the internet. It is the addressing system of the global network.

    • Any machine connected to a network has an IP addres, a unique string of numbers used to identify the device on the network, like the Brazilian CPF! It looks like this: 74.125.19.147
    • Main function: To allow data to be sent from one device to another (computers, serves, routers, mobile phones) through unique addresses called IP adresses;
    • IP works alongside other protocols, especially:
      • TCP (Transmission Control Protocol) → Ensures that data arrives complete and in the correct order.
        • More secure - used for messages, for example. It guarantees that your message will be delivered.
      • DNS (Domain Name System) → Translates names like google.com into IP adresses.
        • Faster - used for calls and voice transmissions, for example. No guarantee that the data will be delivered.
    • It is one of the technical foundations of the internet.

    Currently, there are two versios of the Internet Protocol (IP): IPv4 and IPv6.

    IPv4 is the older version and still widely used. It uses 32-bit addresses, represented by four decimal number between 0 and 255, separated by dots. Such as 192.168.0.1 This structure allows for approximately 4.3 billions distinct addresses. With the rapid expansion of the internet and the growing number of the connected devices, this number has become insufficient.

    IPv6 was created to address this limitation. It uses 128-bit addresses, expressed in hexadecimal notation and separated by colons. Such as 2001:0db8:85a3:0000:0000:8a2e:0370:7334. IPv6 supports aproximately 2¹²⁸ addresses, wich is about 340 undecillion - more than enough to meet current and future demand.

    In addiotion to the expansion od address space, IPv6 introduces other technical improvements:

    • Automatic configuration: in IPv4, a DHPC (Dynamic Host Configuration Protocol) server is usually required to assign addresses to devices. n IPv6, the stateless autoconfiguration (SLAAC) function allows the device itself to generate its own address automatically, reducing reliance on external servers and minising network traffic.
    • Security: IPv6 was designed with native support for IPsec, a suite of protocols for authentication and encryption of data. Something optional and less standardised in IPv4.
    • More effecient routing: the structure of IPv6 packets has been simplifield, allowing faster data processing by routers and better performance in complex networks.

    A website is a set of files written in languages such as HTML, CSS and JavaScript, organised on a server, wich the browser interprets and displays to the user.

    ⎆ Coded files that are interpreted by browsers

    Languages that make up these files

    • HTML - Defines the structure of elements (text, headings, links)
    • CSS - Defines apparence (colours, fonts, layout)
    • JavaScript - Defines behavious and interactions (menus, forms, animations)

Through the Internet, you access the Web via browsers, wich are responsible for requesting and displaying the information served/returned by servers. For example, torrents, instant messaging, and apps are other services that use the Internet's infrastructure but not necessarily the Web!❞ - Progra{M}aria

    Simplifying

    When we search using browsers, send emails, or access recipes, we are requesting data stored on remote servers. These servers send us files composed of code (HTML, CSS, JavaScript, etc.), wich are interpreted by browsers and displayed as pages, text, images or videos.

    Berfore reaching us, this data travels through a network of servers and routers, passing through various points until it reaches our device.

    Where is this data stored?

    On servers locatde in datacentres around the world. Websites, emails, videos, searches, all are stored on these servers.

    What happens when you search for something?

    1. The browser sends a request to a server - Google's, for example;
    2. This request is routed through the internet - passing through several devices (routers, services, cables);
    3. When it reaches the correct server, it responds with files containing the search content;
    4. These files, composed of programming code, are interpreted by the browser, which then display the content - the visual result.

    HTML ✶⋆.˚

    Introduction to HTML

    What is it?

    HTML – HyperText Markup Language

    HTML is the language used to structure the content of web pages. It defines the hierarchy and organisation of elements such as text, heading, lists, links, images, videos, and tables. It is the skeletonof the page.

    It's important to note that HTML is not a programming language, but a markup language.

    What is hipertext?

    Hypertext refers to a system of organising information in which documents are connected through links. These links allow the user to navigate from one part of a document to another, or between different documents entirely, without following a linear sequence.

    In practical terms, hypertext enables a web page to contain not only plain text, but also other media like images, videos, and audio, all connected through hyperlinks . This allows for non-linear reading and dynamic, interactive user experience.

    Technically, a hypertext document is just a text that contains embedded links. HTML is the standard language used to created such documents for the web. It allows authors to structure content and link it to other pages, making it the foundation of web navigation.

    How does the structure of an HTML document work?

    An HTML document is organised in a hierarchial structure, which defines how elements relate to each other and are displayed on the page. The basic structure of an HTML document includes the following parts:

    1. Document type declaration: <!DOCTYPE html> - informs the browser that this is an HTML document.
    2. Root element: <html> - wraps all the content of the page.
    3. Head element: <head> - contains metadata such as the title, links to stylesheets, and scripts.
    4. Body element: <body> - contains the visible content of the page, such as text, images, and link.

    Example of a basic structure:

        <!DOCTYPE html>
        <html>
            <head>
                <title>Page title</title>
                <link rel="stylesheet" href="styles.css">
            </head>
            <body>
                <h1>Welcome to my website!</h1>
                <p>This is a sample paragraph.</p>
                <a href="https://www.example.com">Click here</a>
            </body>
        </html>

    This structure is the foundation of any HTML page. From it, you can add more elements and customise the content.

    HTML Tags

    What are they?

    HTML tags are fundamental elements of the language. They act like "labels"that indicate the role of each part of the content on a page - headings, paragraphs, lists, links, images, tables, among others.

    Each tag is written between "less than" and "greater than" sings (< >). Most tags have an opening and a closing:

    <p>Desired text</p>

    In the example above:

    • <p> opens a paragraph;
    • </p> closes the paragraph.

    The slash / insides the tag indicates the end of that element.

    Not all tags require a closing. Some are called self-closing tags, such as <img> (image) and <br> (line break). These do not have internal content and used on their own.

    HTML is made up almost entirely of tags, so it is essencial to know them well. For a full list:

    • HTML elements reference

    Main Tags and Their Functions

    Some of the main HTML tags you should know are:

    • <h1> to <h6>: Headings, from most important (h1) to least important (h6).
    • <p>: Paragraph.
    • <a>: Link (anchor).
    • <img>: Image.
    • <ul>, <ol>, <li>: Unordered lists (ul), ordered lists (ol), and list items (li).
    • <div>: Division or generic section.
    • <span>: Span, used to style specific parts of text.
    • <br>: Line break.
    • <table>, <tr>, <td>, <th>: Table, table row, table cell, and table header.

    These tags are fundamental for structuring the content of an HTML page. You can combine and nest them to create complex layouts and organise information clearly.

    Sematic Elements

    What They Are and Why They Matter

    Sematic elements are HTML tags that clearly describe the type of content they contain, going beyond mere visual structuring. They indicate the purpose and function of the content to browsers, screen readers, search engines, and other automated agents.

    The importance of semantic elements lies in several aspects:

    • Accessibility: screem readers and assistive devices interpret the content more effectivelly, providing more inclusive navigation for people with disabilities;
    • SEO (Search Engine Optimisation): search engines use semantics to understand the hierarchy and relevance of information, improving ranking in results;
    • Code maintenance and readability: makes the code more organised, readable, and easier for other developers to maintain;
    • Future compatibility: ensures the side follows modern web standards, making it easier to adapt to new technologies and browsers.

    Therefore, the correct use of semantic elements is essential for creating efficient, accessible, and optimised web pages.

    Main Examples:

    • <header>: Page or section header.
    • <nav>: Navigation, links to other pages or sections.
    • <main>: Main content of the page.
    • <article>: Independent article, such as blog post.
    • <section>: Generic content section.
    • <aside>: Related content, such as sidebar.
    • <footer>: Page or section footer.

    HTML Attributes

    What are HTML Attributes?

    HTML attributes are additional information that can be added to tags to modify their behaviour or provide extra details. They are written inside the opening tag, after the tag name.

    Example:

        <a href="https://www.example.com" target="_blank">Click here</a>

    In the example above, the <a> (link) tag has two attributes:

    • href: Defines the link's address;
    • target: Specifies where the link will open (in this case, in a new tab).

    What Are the Main Atributes?

    Some of the main HTML attributes you should know are:

    • id: Unique identifier for the element;
    • class: CSS class for styling;
    • style: Inline style for the element;
    • src: Source of an image or script;
    • alt: Alternative text for images;
    • href: Address of a link;
    • target: Specifies where the link will open (e.g. _blank to open a new tab);
    • title: Descriptive text displayed when hovering over the element;
    • lang: Defines the content language;

    These attributes are widely used for styling, accessibility, and functionality of HTML elements. You can combine them as needed to meet the requirements of your project.

    More Details

    Link:

    • Tag used to link an external resourse to the HTML
    • It has no closing tag (</link>) because it is a self-contained (self-closing)
    • It's not only for CSS - it is also used for icons, external fonts, etc.

    rel Attribute:

    • Means "Relationship"
    • Specifies the relationship between the HTML and the linked file.
    • Example: rel="stylesheet” → indicates that external files is a CSS stylesheet.

    href Attribute:

    • Stands for "hyperlink reference"
    • Points to the location of the CSS file to be used.
    • Example: href:"style.css” → the browser will look for a file called style.css in the same folder as the HTML file.

    Stylesheet:

    • Tell the browser: "the file I'm linking is a CSS style sheet". That is, the browser understands that the content of this files should be used to styles the HTML page (colours, donts, sizes, etc.)
    • IMPORTANT! If you replace stylesheet with any other word, the browser will not aplly the CSS because it will not recognise the type of relationship.

    target Attribute:

    • Specifies where the link should open
    • Example: target="_blank" → opens the link in a new browser tab.

    Simplifying:

    • link : Tag used to link an external resource to HTML;
    • stylesheet : CSS style file;/li>
    • rel : Defines the relationship between HTML and the external file;
    • href : Path to the CSS file;
    • target : Where the link will open.

    Best Pratices for Writing HTML

    Writing clean and organised HTML code is essencial to facilitate maintenance and understanding pf the project. Moreover, it ensures your site is accessible and properly interpreted by browsers and search engines.

    Some good pratices include:

    • Using semantic tags whenever possible, to improve accessibility and content comprehension;
    • Organising the code with proper indentation to aind readability;
    • Closing tags correctly to avoid rendering issues;
    • Section-based organisation;
    • Using comments to explain parts of the code, if necessary;
    • Avoiding excessive use of inline tags, preferring classes and IDs for styling;
    • Using alt attributes in images to improve accessibility;

    Code Organisation

    Well-organised code makes it easier to read, maintain and collaborate with other. Structure elements ins a clear hierarchy, group related content, and avoid unnecessary duplication. Each section od the page should have a specific and weel defined function, with appropriate use of <div>, <section>, <article>, among others.

    Proper Use of Semantic Tags

    Avoid using <div> ou <span> for everything. Prefer semantic elements like <header>, <nav>, <main>, <section>, <article>, <aside> e <footer>. These inform the browser and other agents about the role of each piece of content, improving accessibility, SEO, and the site's structure.

    Indentation

    Proper indetation, with two or four spaces per level, helps visualise the hierarchy of elements. All child content should be indented relative to its parent element. This makes the code more readable and reduces the likelihood of tag-closing errors.

    Example:

    <ul>
  <li>Item 1</li>
  <li>Item 2</li>
</ul>

    Note: Well-written and organised code improves SEO (Search Engines Optimization), as search engines can better understand the page content. If your code contains errors, the browser may not render the page correctly or may even penalise you in search rankings.

    CSS ✶⋆.˚

    Introduction CSS

    What is CSS?

    CSS – Cascading Style Sheets

    CSS is the language used to style and format the content of HTML pages. It defines how elements should be displayed, including colours, fonts, spacing, and layout. It is responsible for the visual appearance of the page.

    important: CSS is not a programming language either. It is a styling language, used solely to define the visual presentation of content.

    How does it work?

    CSS work by applying styling rules to HTML elements. These rules are made up of selectors and declarations, which determine which elements are styled and how.

    Example of a CSS rule:

            h1 {
            color: blue;
            font-size: 24px;
        }

    In the example above:

    • h1 is the selector, indicating that the rule applies to all <h1> elements;
    • color: blue; is a declaration that sets the text color to blue;
    • font-size: 24px; is another declaration that sets the font size to 24 pixels.

    CSS rules can be applied to any HTML element, allowing you to customise the appearence of text, images, links, tables, and much more.

    The most common CSS properties include:

    • color;
    • background-color;
    • font-size;
    • font-family;
    • margin;
    • padding;
    • border;
    • width e height;

    How to include it in an HTML document?

    Adding CSS to an HTML document can be done in three main ways:

    1. Inline: Adding styles directly within the HTML tah using the style attribute.
      2.         <h1 style="color: blue; font-size: 24px;">Title</h1>
    2. Internal: Including a block of styles inside the HTML <head> tag.
      3.         <head>
            <style>
                h1 {
                    color: blue;                    
                    font-size: 24px;
                }
            </style>
        </head>
    3. External: Creating a separate CSS file and linking it to the HTML.
      4.         <head>
            <link rel="stylesheet" href="styles.css">
        </head>

    The most recommended method is the external one, as it allows you to separate content (HTML) from presentation (CSS), making it easier to maintain and reuse styles across multiple pages.

    Selectors

    CSS selectors are used to select which HTML elements will be styled. These are several types of selectors, each with its own purpose:

    • Type selectors Select all elements of given type (e.g., h1, p, div).
    • Class selectors: Select elements with a specific class (e.g., .my-classe).
    • ID selectors: Select an element with a specific ID (ex: #my-id).
    • Attributes selectors: Select elements with a specific attribute (e.g., [type="text"]).
    • Combined selectors Combine different selectors for greater precision (e.g., div.my-classe, h1#my-id).

    Example of selector usage:

        p {
        color: red;
    }
    .my-classe {
        font-size: 20px;
    }
    #my-id {
        background-color: yellow;
    }

    These selectors are used in HTML and later specified in CSS for styling.

    Important: An ID must be unique on a page, whereas classes can be reused across multiple elements.

    Units of measurement

    CSS uses different units of measurement to define sizes, spacing, and proportions. The most common ones are:

    • px: Pixels, a fixed unit based on screen resolution;
    • em: Unit relative to the font size of the parente element;
    • rem: Unit relative to the font size of the root element (html);
    • %: Percentage, relative to the size of the parent element;
    • vh: Viewport height, 1% of the height of the viewport window;
    • vw: Viewport width, 1% of the width of the viewport window.

    Example of unit usage:

        h1 {
        font-size: 2em;
        margin: 10px;
    }
    .container {
        width: 80%;
        height: 50vh;
    }

    Colours

    Colours in CSS can be defined in several ways:

    • Colour names:
      • Limitations: There are only 147 standard names.
      • No need to memorise, but not very flexible.
      • Pratical use: Good for quick teste, but rarely used in professional projects.
      • Example: red, blue, green, black, white, gray, orange, etc.
    • Hexadecimal codes:
      • Format: # followed by 6 hexadecimal digits (0-9, A-F);
      • Widely used in professional projects.
      • Example: #FF0000 (red), #0000FF (blue);
    • RGB:
      • Format: rgb(amount of red, amount of green, amount of blue);
      • Each value ranges from 0 to 255;
      • Allows for a wide range of colours;
      • Example: rgb(255, 0, 0) is pure red, rgb(0, 255, 0) is pure green.
    • RGBA:
      • Similar to RGB, but with a fourth value for opacity (0 to 1);
      • Allows you to set the transparency of the colour;
      • Example: rgba(255, 0, 0, 0.5) is red with 50% opacity.
    • HSL:
      • Format: hsl(hue, saturation, lightness);
      • Hue varies from 0 to 360 degrees (colour wheel);
      • Saturation and lightness range from 0% to 100%;
      • Example: hsl(0, 100%, 50%) is pure red, hsl(240, 100%, 50%) is pure blue.
    • HSLA:
      • Similar to HSL, but with a fourth value for opacity (0 to 1);
      • Allows you to set the transparency of the colour;
      • Example: hsla(0, 100%, 50%, 0.5) is red with 50% opacity.

    Example of using colours in CSS:

        body {
        background-color: #f0f0f0;
    }
    h1 {
        color: rgb(255, 0, 0);
    }
    p {
        color: rgba(0, 0, 255, 0.5);
    }

    Box Model

    The CSS box model is a fundamental concept that defines how HTML elements occupy space on the page. Every rendered element on a website is, in practice, a "box"made up of four layers:

    1. Content — the actual content (text, image, etc.);
    2. Padding — the inner space between the content and the border of the box;
    3. Border — the border of the box (visible or not);
    4. Margin — the outer space between the box and other elements.

    These four parts form the visual structure and behaviour of the element in relation to the layout.

    The structure of an element in the box model is as folllows:

     
╔════════════════════════════════════════╗  ← margin
║  ┌──────────────────────────────────┐  ║
║  │          padding                 │  ║
║  │  ┌────────────────────────────┐  │  ║
║  │  │         content            │  │  ║
║  │  └────────────────────────────┘  │  ║
║  └──────────────────────────────────┘  ║
╚════════════════════════════════════════╝  ← border

    Padding moves the content away from inner border. Margin on the oher hand, separates the element from other elements on the page. The border can have colour, thickness, and style, or it may not appear at all, but it always exists logically.

    And to style these elements, we use CSS properties as shown in the example below:

    .box {
  width: 200px;
  padding: 20px;
  border: 2px solid black;
  margin: 30px;
}

    In this example, the element will have:

    • 200px of width for the content only;
    • 20px of inner space (padding);
    • A 2px border around the box;
    • 30px of distance from external elements (margin).

    JavaScript ✶⋆.˚

    Introduction to JS

    What is JavaScript?

    JavaScript is an intrepreted, object-oriented programming language mainly used in web development to make pages interactive and dynamic. Alongside HTML and CSS, it forms the foundation for building interfaces on the web.

    It was originally created to run in the browser and allow HTML pages to respond to user actions, such as clicks, form inputs, animations, and real-time content changes.

    Without JavaScript, a page is purely static, meaning it only displays content but does react to anything.

    What is it used for?

    JavaScript is used to add behaviour to web pages. With it, the following is possible:

    • Manipulate HTML elements and change their content, appearance, or structure (DOM);
    • Respond to user events (such as clicks, scrolling, or typing);
    • Validate forms before they are sent to the server;
    • Control the programme flow using conditions and loops;
    • Temporaarily store data in the browser (g.g., localStorage);
    • Make requests to servers without reloading the page (AJAX);
    • Create animations and dynamic.

    How JS work in the browser

    Where is JS executed?

    In the context of the web, JavaScript is executed directly by the user's browser (such as Chrome, Firefox, or Edge). These browsers have built-in engines - called "JavaScript engines" (such as V8 in Chrome) - which interpret and run the code in real time.

    This means JS does not require installation or prior compilation to work: a browser is enough. When a page containing JavaScript is opened, the browser processes the code and executes it automatically.

    Today, JavaScript can also run outside the browser (such as with Node.js), but it's most common use is still on the front-end.

    Loading and running JavaScript

    JavaScript works by interpreting and executing code directly in the user's browser. It can be included in an HTML document in the three main ways:

    1. Inline: Adding JS code directly in an HTML tga using the onclick attribute, for example.
      2.     <button onclick="alert('Hello!')">Click here</button>
    2. Internal: Including a script block within the <head> or <body> tag of the HTML.
      3.     <head>
        <script>
            function showAlert() {
                alert('Hello!');
            }
        </script>
    </head>
    3. External: Creating a separate JS file and linking it to the HTML.
      4.     <head>
        <script src="script.js"></script>
    </head>

    The most recommended approach is the external one, as it allows separating the content (HTML) from the logic (JavaScript), making the code easier to maintain and reuse.

    Interaction with HTML and CSS

    JavaScript connects to HTML and CSS through the DOM (Document Object Model), a tree-like representation of the page structure.

    With it, you can::

    • Select element on the page (`document.querySelector`, `getElementById`, etc.);
    • Change content and attributes (`element.innerText`, `element.setAttribute`);
    • Modify styles directly(`element.style.color = "red"`);
    • Add or remove CSS classes;
    • Insert, move or remove elements from the page.

    Variable and data types

    In JS, variables are used to store values and can be declared using the keywords var, let, or const. Each has its own characteristics:

    • var: Declares a variable with global or function scope. Can be reassisned and redeclared.
    • let: Declares a variable with block scope. Can be reassigned, but not redeclared in the same scope.
    • const: Declares a constant, whose value cannot be reassigned. Also has block scope.

    Example of variables declaration:

        var name = 'João';
    let age = 30;
    const pi = 3.14;

    The most common data types in JavaScript include

    • String: Text, enclosed in single or double quotes (e.g., 'Hello', 'Word');
    • Number: Integers or decimals (e.g., 42, 3.14);
    • Boolean: Logical values true or false;
    • Array: List of values, enclosed in square brackets (e.g., [1, 2, 3], ['apple', 'banana']);
    • Object: Key-values structure, enclosed in curly braces (e.g., { name: 'João', age: 30 });
    • Null: Null value;
    • Undefined: Underfined vallue.

    Operators

    Arithmetic Operators

    Used to perform mathematical operations:

    • + (addition)
    • - (subtraction)
    • * (multiplication)
    • / (division)
    • % (modulo, remainder of division)

    Comparison Operators

    Used to compare values and return either true or false:

    • == (equality)
    • != (inequality)
    • === (strict equality, compares value and type)
    • !== (strict inequality, compares value and type)
    • > (greater than)
    • < (less than)
    • >= (greater than or equal to)
    • <= (less than or equal to)

    Logical Operators

    Used to combine boolean expressions:

    • && (logical AND, returns true if both expressions are true)
    • || (logical OR, returns true if at least one expression is true)
    • ! (logical NOT, inverts the bollean value)

    Control Structures

    Control structure are used to control the flow of code execution, allowing decisions and repetition. The main control structures in JavaScript are:

    • if...else: A conditional that executes one block of code if a condition is true, and another block if it is false.
    • switch:A conditional structure that compares an expression with multiple possible cases.
    • for:A loop that iterates over a block of code a specific number of times.
    • while:A loop that runs a block of code while a condition is true.
    • do...while:Similar to while, but guarantees the block will run at least onde before checking the condition.
